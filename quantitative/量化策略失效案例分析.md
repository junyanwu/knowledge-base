# 量化策略失效案例分析

## 1. 2015年A股股灾案例

### 1.1 事件概述

**时间：** 2015年6月15日 - 8月26日  
**跌幅：** 上证指数下跌约45%  
**熔断：** 千股跌停常态化

### 1.2 量化策略表现

| 策略类型 | 股灾前年化 | 股灾期间回撤 | 恢复时间 |
|---------|-----------|------------|---------|
| CTA趋势 | 20-30% | -15~-25% | 6个月 |
| 市场中性 | 10-15% | -20~-30% | 12个月 |
| 指数增强 | 5-10% | -40~-50% | 18个月 |
| 多因子 | 15-20% | -25~-35% | 9个月 |

### 1.3 失效原因

**1. 流动性危机**
```python
# 正常情况：可以正常买卖
liquidity_ratio = volume / avg_volume  # 约1.0

# 股灾情况：流动性枯竭
liquidity_ratio = volume / avg_volume  # <0.3
# 结果：无法卖出，止损失效
```

**2. 相关性崩溃**
```python
# 正常情况：分散化有效
correlation_matrix = {
    'stock1-stock2': 0.3,
    'stock1-stock3': 0.4,
    'stock2-stock3': 0.35
}

# 股灾情况：相关性趋近1
correlation_matrix = {
    'stock1-stock2': 0.95,
    'stock1-stock3': 0.92,
    'stock2-stock3': 0.97
}
# 结果：分散化失效
```

**3. 模型失效**
- 历史数据无法预测极端事件
- 风险模型低估尾部风险
- VaR严重低估实际损失

### 1.4 教训与改进

**改进措施：**

1. **流动性管理**
```python
class LiquidityManager:
    def __init__(self, max_illiquid_assets=0.3):
        self.max_illiquid = max_illiquid_assets
    
    def check_liquidity(self, portfolio):
        illiquid_ratio = portfolio['illiquid'] / portfolio['total']
        if illiquid_ratio > self.max_illiquid:
            return False, "流动性资产不足"
        return True, "OK"
```

2. **相关性监控**
```python
def monitor_correlation(returns, threshold=0.8):
    """监控相关性"""
    corr_matrix = returns.corr()
    avg_corr = corr_matrix.values[np.triu_indices(len(corr_matrix), k=1)].mean()
    
    if avg_corr > threshold:
        return "WARNING: 高相关性，分散化失效"
    return "OK"
```

3. **尾部风险保护**
```python
def tail_risk_protection(portfolio, var_limit=0.15):
    """尾部风险保护"""
    var_99 = portfolio.pct_change().quantile(0.01)
    
    if abs(var_99) > var_limit:
        return "WARNING: 尾部风险过高"
    return "OK"
```

## 2. 2020年美股熔断案例

### 2.1 事件概述

**时间：** 2020年3月  
**跌幅：** 道指下跌约37%  
**熔断：** 10天内4次熔断

### 2.2 量化策略表现

| 策略 | 表现 | 原因 |
|------|------|------|
| 风险平价 | -15~-20% | 所有资产下跌 |
| 趋势跟踪 | -10~-15% | 趋势反转过快 |
| 统计套利 | -20~-30% | 相关性崩溃 |
| 高频交易 | +5~+15% | 波动率机会 |

### 2.3 关键事件

**VIX飙升：**
```
正常：VIX = 15-20
危机：VIX = 80+
结果：期权定价失效，波动率策略逆转
```

**国债流动性危机：**
```
3月9日-18日：
- 国债市场流动性枯竭
- ETF大幅折价
- 对冲失效
```

### 2.4 教训

**真正分散化：**
- 需要在不同资产类别间分散
- 现金是最好的对冲
- 避免虚假分散

**极端情况准备：**
- 保持足够现金
- 准备应急方案
- 不要依赖单一模型

## 3. Long-Term Capital Management (LTCM)案例

### 3.1 背景

**成立：** 1994年  
**创始人：** John Meriwether, 诺贝尔奖得主  
**策略：** 固定收益套利  
**杠杆：** 25:1

### 3.2 崩溃

**1998年：**
- 俄罗斯债务违约
- 相关性反转
- 流动性危机
- 损失44亿美元

### 3.3 失效原因

**过度杠杆：**
- 25倍杠杆
- 小波动导致大损失
- 无法平仓

**相关性假设错误：**
- 假设相关性稳定
- 危机时相关性反转
- 对冲失效

**流动性风险：**
- 规模过大
- 无法及时平仓
- 市场恐慌

### 3.4 教训

**杠杆管理：**
```python
def leverage_control(leverage, max_leverage=3):
    """杠杆控制"""
    if leverage > max_leverage:
        return False, "杠杆过高"
    return True, "OK"
```

**相关性风险：**
```python
def correlation_risk(corr_matrix, threshold=0.7):
    """相关性风险监控"""
    avg_corr = corr_matrix.values[
        np.triu_indices(len(corr_matrix), k=1)
    ].mean()
    
    if avg_corr > threshold:
        return False, "相关性过高"
    return True, "OK"
```

## 4. 2022年量化基金回撤案例

### 4.1 背景

**时间：** 2022年  
**原因：** 通胀高企、利率上升  
**影响：** 量化基金普遍回撤

### 4.2 表现

| 基金 | 策略 | 回撤 |
|------|------|------|
| Renaissance Institutional | 市场中性 | -20% |
| Two Sigma | 多策略 | -15% |
| DE Shaw | 多策略 | -18% |
| Citadel Wellington | 多策略 | -8% |

### 4.3 原因

**宏观环境变化：**
- 通胀上升
- 利率上升
- 成长股下跌

**因子失效：**
- 价值因子回归
- 动量因子失效
- 质量因子表现差

**相关性变化：**
- 传统相关性关系反转
- 历史模式失效

### 4.4 教训

**适应性：**
- 模型需要适应宏观环境
- 不能过度依赖历史数据
- 保持策略多样性

**风险管理：**
- 宏观风险监控
- 因子有效性监控
- 动态调整

**2. 相关性崩溃**

- 对冲逆转

**3. 模型失效**
- 历史数据失效
- 极端事件无法预测
- 过度依赖模型

**4. 杠杆风险**
- 小波动大损失
- 强制平仓
- 踩踏

**1. 流动性管理**
- 保持充足现金
- 控制持仓集中度
- 避免流动性差的资产

**2. 风险分散**
- 真正分散化
- 跨资产类别
- 跨市场

**3. 杠杆控制**
- 适度杠杆
- 动态调整
- 极端情况准备

**4. 模型监控**
- 持续监控
- 样本外测试
- 极端情况预案

### 最佳实践

**风险管理优先：**
```
流动性风险 > 杠杆风险 > 模型风险 > 操作风险
```

**准备极端情况：**
- 保持现金储备
