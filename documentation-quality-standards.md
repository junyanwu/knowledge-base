# 学习内容质量优化标准

> 📊 **质量优先** - 深度分析 > 浅层总结，数据验证 > 理论陈述

---

## 一、质量评估维度

### 1.1 深度评分 (1-5 分)

| 分数 | 标准 | 示例 |
|------|------|------|
| 1 分 | 浅层总结，复述原文 | "研报说..." |
| 2 分 | 有归纳，缺少分析 | "主要观点是..." |
| 3 分 | 有分析，缺少验证 | "可能原因是..." |
| 4 分 | 深度分析 + 数据验证 | "数据显示...，验证了..." |
| 5 分 | 批判性分析 + 实操建议 | "逻辑成立，但...，建议..." |

**目标**: 所有产出≥4 分

---

### 1.2 数据支撑 (必须有)

**必备数据**:
- ✅ 具体数值 (增长率/收益率/估值等)
- ✅ 时间范围 (2021-2026 等)
- ✅ 数据来源 (Wind/统计局等)
- ✅ 对比基准 (沪深 300/行业平均等)

**示例**:
```
❌ 差：新能源行业增长很快
✅ 好：2021-2023 年光伏装机量 CAGR +34%，
      远超行业平均 +18% (来源：CPIA)
```

---

### 1.3 实操性 (必须有)

**必备内容**:
- ✅ 可执行的策略/方法
- ✅ 具体参数/阈值
- ✅ 代码实现 (如适用)
- ✅ 风险提示

**示例**:
```
❌ 差：可以用动量策略
✅ 好：动量策略参数：
      - 回看周期：60 日
      - 调仓频率：周度
      - 止损：-8%
      - 回测年化：+15.3%
```

## 四、质量提升方法

### 4.1 深度分析技巧

**5Why 分析法**:
```
现象：新能源 ETF 2024 年下跌 22%
Why1? → 产能过剩
Why2? → 投资过热，2023 年产能 +80%
Why3? → 政策补贴过度
Why4? → 地方政府招商引资冲动
Why5? → GDP 考核机制

根因：政策导向扭曲
```

**对比分析法**:
```
横向对比：
- 中国光伏 vs 美国光伏 vs 欧洲光伏

纵向对比：
- 2021 年 vs 2022 年 vs 2023 年

基准对比：
- 行业平均 vs 龙头 vs 尾部
```

### 4.2 数据验证方法

**多源验证**:
```
数据点：2023 年光伏装机 350GW

验证 1: CPIA → 350GW ✅
验证 2: IEA → 345GW ✅
验证 3: BNEF → 355GW ✅

结论：数据可靠
```

**历史回测**:
```
策略：动量策略

回测 1: 2015-2019 → +12%/年
回测 2: 2020-2024 → +15%/年
回测 3: 2015-2024 → +13.5%/年

结论：策略稳定
```

## 五、质量评分示例

### 示例 1: 差 (2 分)

```
新能源行业前景很好，建议配置。
光伏和储能都是好赛道。
```

**问题**:
- ❌ 无数据支撑
- ❌ 无逻辑分析
- ❌ 无实操建议

---

### 示例 2: 中 (3 分)

```
2021-2023 年光伏装机量增长很快，
CAGR 达到 34%。
建议配置光伏 ETF，长期持有。
```

**问题**:
- ⚠️ 有数据，但无来源
- ⚠️ 无估值分析
- ⚠️ 无风险提示

---

### 示例 3: 好 (5 分)

```
2021-2023 年光伏装机量 CAGR +34%
(来源：CPIA)，远超行业平均 +18%。

核心驱动：
1. 碳中和政策 (2030 碳达峰)
2. 度电成本下降 60% (2015-2023)
3. 储能配套政策完善

估值分析：
- 当前 PE 25x，历史中位数 30x
- PEG 0.8，低于成长股平均 1.2

实操建议：
- 配置：光伏 ETF (515790) 20%
- 入场：PE<28x 时分批买入
- 止损：-15% 或 PE>40x
- 目标：+30-50%/年

风险提示：
- 产能过剩风险 (概率 40%)
- 政策退坡风险 (概率 20%)
- 对冲：配置储能 ETF 分散
```

**优点**:
- ✅ 数据充分 + 来源
- ✅ 逻辑完整 + 验证
- ✅ 实操建议 + 风控

## 六、持续改进

### 每次产出后反思

1. **数据够吗？**
   - 是否有关键数据缺失？
   - 数据来源可靠吗？
   - 有交叉验证吗？

2. **逻辑对吗？**
   - 因果关系成立吗？
   - 有选择性偏差吗？
   - 有反例吗？

3. **能用吗？**
   - 有具体参数吗？
   - 有代码实现吗？
   - 有风险提示吗？

4. **够深吗？**
   - 停留在表面吗？
   - 有批判性分析吗？
   - 有改进建议吗？
